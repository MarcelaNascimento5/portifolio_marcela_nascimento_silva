# 🚀 Batalha de Modelos & Engenharia de Prompt (XML)

## 📝 Descrição do Projeto
Este projeto apresenta uma análise comparativa de desempenho entre diversos modelos de **Inteligência Artificial** utilizando técnicas de **Engenharia de Prompt**[cite: 1, 2]. [cite_start]O objetivo central foi avaliar a capacidade de diferentes LLMs em interpretar um **Prompt Estruturado em XML** para gerar uma página HTML *Single Page* com CSS integrado[cite: 4, 6].

Desenvolvido para a disciplina de **Engenharia de Prompt e Aplicações em IA**, o experimento submeteu o mesmo protocolo técnico a ferramentas como ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca[cite: 1, 5].

## 🚀 Tecnologias e Modelos Testados
* **Estruturação:** XML (tags delimitadoras para maior precisão semântica).
* **Tecnologias Alvo:** HTML5 e CSS3.
* **Modelos Avaliados:** Claude, Gemini, DeepSeek, GPT, Qwen, Grok e Maritaca.

## 📊 Resultados e Aprendizados
O projeto revelou disparidades significativas na compreensão de prompts estruturados entre as IAs:

* **Destaque de Performance:** O **Claude** demonstrou a maior compreensão da estrutura XML, superando as expectativas ao adicionar funcionalidades extras e história ao conteúdo[cite: 8, 13].
* **Eficiência de Tokens:** O modelo **Qwen** destacou-se pela verbosidade reduzida, utilizando significativamente menos tokens (350-400) para entregar resultados similares[cite: 9, 15].
* **Qualidade Técnica:** O **DeepSeek** apresentou resultados precisos, com excelente geração de imagens e ausência de bugs de sintaxe[cite: 8, 9].
* **Prototipagem vs. Complexidade:** Concluiu-se que o **Claude** é a melhor escolha para códigos complexos, enquanto o **Gemini** é ideal para prototipagem rápida e de qualidade[cite: 17].

## 🔧 Estrutura do Prompt (Protocolo XML)
[cite_start]O prompt foi organizado sob as seguintes tags principais[cite: 6]:
* `<tarefa>`: Definição da atividade e objetivo.
* `<diretrizes_design>`: Regras para layout, cores e tipografia.
* `<obrigatoriedades_tecnicas>`: Itens essenciais como menu de navegação e portfólio.
* `<metrica_obrigatoria>`: Instrução para estimativa de tokens gerados.
